Biography

Provided Queries:

This section contains links to pre-written sparql queries. Most of the queries below appear as:

PREFIX cwrc: < http://sparql.cwrc.ca/ontologies/cwrc#>
PREFIX rdf: < http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX cwrc: < http://sparql.cwrc.ca/ontologies/cwrc#>
PREFIX rdf: < http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: < http://www.w3.org/2000/01/rdf-schema#>
PREFIX data: < http://cwrc.ca/cwrcdata/>
PREFIX skos: < http://www.w3.org/2004/02/skos/core#>
PREFIX oa: < http://www.w3.org/ns/oa#>
SELECT ?obj ?snippet WHERE
    GRAPH < http://sparql.cwrc.ca/db/BiographyV2Beta> {
    ?context ?pred ?obj;
        cwrc:contextFocus ?person;
        oa:hasTarget ?target;
        rdf:type cwrc:FamilyContext.
    ?person rdfs:label "Fiennes, Celia".
}.
?target oa:hasSelector ?xpath_selector.
?xpath_selector oa:refinedBy ?text_selector.
?text_selector oa:exact ?snippet.
?pred cwrc:subjectCentricPredicate cwrc:hasFather.
}

This query starts with listing the prefixes of all the ontolgies used and others commonly used. The varibales return are the object we're searching for and a snippet of how the data is stored. This query also shows the graph that is being searched and specifies the context, which is listed after rdf:type. Outside of the graph section, the query searches for a snippet of text that the data is stored as. The last line concerns what we're searching for - it allows us to specify the relationship between the subject and the object.

The name of the person whose information is being searched is in bold. Here the person is Celia Fiennes. If you want to search for another person you can find their name on http://orlando.cambridge.org

The other data in bold is the predicate that the subject and object are connected by, there are many predicates on the cwrc website and you can search for the specific one you need.

Family & Friends & Intimate Relationships

Notice the queries above only differ slightly. They each specify a different person and have a different predicate on following cwrc:subjectCentricPredicate, but are otherwise identical. The predicate following 'cwrc:' can be replaced with a many others which are listed on the cwrc website. Some examples include:

  • hasAunt
  • hasUncle
  • hasGrandFather
  • hasGrandMother
  • hasGrandDaughter
  • hasGrandSon
  • hasNephew
  • hasStepSister
  • hasStepBrother

Location

How many authors lived in London?

You will notice something different about the query above. Most of the other queries had the following construction under GRAPH:

?context ?pred ?obj

This query had specified the object instead of leaving it as a variable. The object was taken from the GeoNames ontology, if you want to specify a different location then then search for a place on https://www.geonames.org/v3/and replace the number in the URI with the the number in the URI of the location you want.

?context ?pred <http://sws.geonames.org/2643743/>

Occupation

How many authors were also teachers?

Notice how the query above returned a small number as the people who were also teachers. There were certainly more than 45 teachers -- so why was the result so small. This has to do with the specifics of your query, in the data teachers are not only grouped by hasOccupation but also by hasPaidOccupation, hasVolunteerOccupation, etc.

Using hasPaidOccupation and hasVolunteerOccupation alone would look like the following:

How many authors were paid teachers
How many authors were volunteer teachers

You can replace hasOccupation by these other predicates to search for them. However, all these predicates are also subproperties of of the predicate occupation, and you can search for the subproperties by using the rdfs:subPropertyOf* constraint with a star next to it and get a result for everyone who was a teacher. The query below is an example of this:

How many authors were also teachers? (Using subproperties)
